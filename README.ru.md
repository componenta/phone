# Componenta Phone

Объект значения телефонного номера на базе `giggsey/libphonenumber-for-php`.

Используйте его, когда телефонные номера нужно валидировать, нормализовать в E.164, форматировать для интерфейса и сериализовать как стабильную строку.

## Установка

```bash
composer require componenta/phone
```

## Связанные пакеты

Пакет самостоятельный, но опирается на `giggsey/libphonenumber-for-php` для разбора и форматирования номеров.

| Пакет | Зачем может использоваться рядом |
|---|---|
| `componenta/validation` | Валидирует сырой номер до создания `Phone` в DTO или команде. |
| `componenta/auth` | Может использовать телефон в сценариях OTP или входа без пароля. |
| `componenta/cqrs` | Команды регистрации, обновления профиля и OTP могут типизировать телефон через этот объект значения. |

## Использование

```php
use Componenta\Stdlib\Phone;

$phone = Phone::create('916 123-45-67', region: 'RU');

(string) $phone;          // "+79161234567"
$phone->toInternational();
$phone->toNational();
$phone->toRfc3966();
$phone->countryCode();    // 7
$phone->detectedRegion(); // "RU"
```

## Конструкторы

- `Phone::create($phone, $region = 'RU')` разбирает и валидирует исходную строку.
- `Phone::fromE164($e164)` создает объект из заранее нормализованной строки E.164 и все равно валидирует ее.
- `Phone::tryCreate($phone, $region = 'RU')` возвращает `null` вместо исключения.

Регион используется только как подсказка для разбора локальных форматов. Номера, начинающиеся с `+`, разбираются без привязки к региону. Сохраненное значение всегда находится в формате E.164.

## Валидация и ошибки

Некорректный ввод приводит к `InvalidArgumentException`. Статический метод `isValid()` подходит для булевых проверок, когда объект значения не нужен.

## Сериализация

`__toString()` и `jsonSerialize()` возвращают значение E.164. `masked()` сохраняет код страны и последние четыре цифры для логов и интерфейса.

## Производительность

Внутренний экземпляр `PhoneNumber` создается лениво и кешируется внутри объекта значения после первого вызова форматирования или классификации.
